## Overview

Apex Utility is a static website for industrial services, built to present a company presence in a clean and organized way. The project relies on HTML together with shared CSS and JavaScript so the same look and behavior carry across the full site. That structure makes it a practical fit for showcasing services, products, and project work from a single web presence.

It is aimed at teams that want a professional website without introducing a heavier application framework. The layout supports consulting-oriented content, product pages, and a contact path that includes WhatsApp prefill, helping visitors move from exploration to direct inquiry with less friction.

## How to Use It

Treat the site as the front-end presentation layer for an industrial utility consultancy or another service-based business.

Typical workflow:

1. Update the service pages with your company details.
2. Edit the product catalogue and project portfolio content.
3. Configure the contact form fields and WhatsApp message text.
4. Deploy the static files to your hosting provider.
5. Share the public site link with visitors and clients.

Since this is a static project, edits are typically made directly in the HTML, CSS, or JavaScript files.

## Configuration

Most of the setup lives in the page source instead of an external configuration system. For customization, check the following areas:

- navigation links in the HTML files
- contact form fields and WhatsApp prefill text
- shared styles in the CSS assets
- page behaviors in the JavaScript files

Example site-level adjustments:

    {
      "contact_channel": "WhatsApp",
      "site_type": "static multi-page website",
      "content_areas": ["services", "products", "projects"]
    }

## FAQ

**How do I change the content?**  
Edit the HTML pages and shared assets directly, then redeploy the site.

**Is it possible to adjust the contact path?**  
Yes. The contact form and WhatsApp prefill can be updated in the relevant page files.

**What should I check if a page breaks or does not open?**  
Review the file paths, linked assets, and navigation references across the multi-page structure.

**How are new updates published?**  
Update the static files and push a new build or deployment.

**Which hosting setup should I use?**  
Any static hosting platform that serves HTML, CSS, and JavaScript will work.
